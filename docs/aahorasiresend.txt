¡Hola! Entiendo perfectamente la frustración. El hecho de que solo se registrara el primer intento es la clave: **Supabase Auth no permite registrar el mismo correo dos veces**, y si el primer intento falló a medias (el usuario se creó en Auth pero no en la tabla `usuarios`), los siguientes intentos fallarán porque el correo ya existe.

### PROTOCOLO DE SOLUCIÓN FINAL:

#### 1. Limpiar usuarios atascados
Ve a tu panel de **Authentication** -> **Users** y **BORRA manualmente** cualquier usuario que aparezca ahí con los correos que estás usando para probar. Esto es vital para que puedas empezar de cero.

#### 2. Ejecutar el Script de Rescate
He creado un script que limpia absolutamente todo el ruido técnico.
1. Ve al **SQL Editor**.
2. Copia y ejecuta todo lo de **docs/AupdateFinal.txt**.
   *Esto borrará cualquier trigger invisible que esté causando el Error 500.*

#### 3. Prueba Maestra (Sin SMTP)
Para confirmar que el login funciona perfecto y que el problema es SOLO el correo de Resend:
1. Ve a **Authentication** -> **Providers** -> **Email**.
2. **DESACTIVA** la opción **Confirm email**.
3. Dale a **Save**.
4. Intenta registrarte en tu página `login.html`.
   - **Si entra directo al directorio:** ¡Felicidades! El código y la base de datos están perfectos. El Error 500 venía solo de la conexión con Resend.
   - **Si falla con Error 500:** Abre la consola del navegador (**F12**), ve a la pestaña **Console** y dime qué error en rojo aparece. He puesto un log especial para capturarlo.

#### Sobre Resend:
Si el login sin confirmación funciona, pero con confirmación falla, es posible que Resend esté rechazando el envío por una cuestión de "Sender not verified" (aunque lo veas verificado) o porque la API Key no tiene permisos de envío. Pero primero hagamos la prueba del Paso 3.

¡Sigue estos pasos y el sistema funcionará de inmediato!
